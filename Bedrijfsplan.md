**Hoofdstuk 5**# Kwaliteit & Security

De Anti-Gravity security-architectuur en kwaliteitsstandaarden.

#### Security als Fundament

Bij LaventeCare zijn veiligheid en kwaliteit geen optionele modules maar de absolute standaard. De 'Anti-Gravity' security-filosofie is ingebakken in elke laag van de architectuur, van firmware tot cloud.

## Security-Architectuur: De 5 Anti-Gravity Wetten

#### Wet 1 **Actief**

*Input is Toxic*

Nooit vertrouw r.Body, URL-parameters of headers. Alles wordt getypeerd, gevalideerd en gesanitized via strict type-checking.

* json.Decoder.DisallowUnknownFields() op alle API endpoints
* Strict Validate() methods op elk request-object
* Parameter sanitization op alle user-facing inputs
* SQL Injection onmogelijk gemaakt via sqlc (geen string concatenatie)

#### Wet 2 **Actief**

*Silence is Golden*

Lekt nooit interne state of stack traces naar de client. Monitoring via interne logs (slog) en Sentry, generieke HTTP 400/500 responses naar buiten.

* Sentry error-tracking voor real-time monitoring
* Structured logging met slog (Go stdlib)
* Generieke foutmeldingen naar clients
* subtle.ConstantTimeCompare voor timing-attack preventie

#### Wet 3 **Actief**

*The Database is a Fortress*

SQL Injection is onmogelijk (sqlc + pgx). Toegang strikt gescheiden per tenant via Row Level Security (RLS) en query-level tenant isolation.

* sqlc-generated queries (zero string concatenation)
* Row Level Security (RLS) op alle kritieke tabellen
* Transaction-scoped tenant context (SET LOCAL)
* Default Deny: tabellen weigeren access zonder valid tenant

#### Wet 4 **Actief**

*Race Conditions are Fatal*

Handlers zijn stateless. Gedeelde resources worden beschermd via sync.Mutex of Channels. Geen race conditions mogelijk.

* Stateless handler design
* Thread-safe shared resources via sync.Mutex
* Atomic operations voor kritieke state changes
* Automated race condition detection in CI/CD

#### Wet 5 **Actief**

*Dependency Paranoia*

Minimale, vertrouwde dependencies. Voorkeur voor standard library en bewezen packages (chi, pgx, sentry-go). Geen bloatware.

* Go stdlib als primaire tooling
* Alleen battle-tested dependencies (chi, pgx)
* Automated vulnerability scanning (Dependabot)
* Regelmatige dependency audit en pruning

## Verdienmodel & Schaalbaarheid

#### Projectontwikkeling (CapEx) **Eenmalig**

Setup- en bouwkosten voor de initiële R&D van IoT-hardware, architectuurontwerp en custom development van webapplicaties.

**Schaalbaarheid:** Lineair — groeit met capaciteit

#### SaaS & Device Licenties (MRR) **Terugkerend**

Abonnementsmodel ('Usage-Based'). Klanten betalen een vast bedrag per maand, per aangesloten IoT-device, of voor toegang tot portalen en dataretentie.

**Schaalbaarheid:** Exponentieel — schaalt met aantal devices/klanten

#### Gelaagd SLA-Model (MRR) **Terugkerend**

Drielaags Service Level Agreement: Basic (€75/mnd — hosting & uptime-monitoring), Professional (€250/mnd — inclusief proactieve patching, dependency updates & 4 uur support/mnd), Enterprise (€500/mnd — geautomatiseerde 24/7 monitoring met priority response, max 4 uur reactietijd binnen kantoortijden, dedicated back-up developer, software escrow). Elke laag biedt hogere marge na initiële setup.

**Schaalbaarheid:** Hoog — pro/enterprise tiers schalen met minimale extra kosten
